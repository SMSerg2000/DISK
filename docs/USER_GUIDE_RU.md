# DISK Diagnostic Tool — Руководство пользователя

## 1. Введение

### Что это за программа

**DISK Diagnostic Tool** — утилита для диагностики жёстких дисков (HDD) и твердотельных накопителей (SSD) под Windows. Аналог легендарной [Victoria HDD](https://hdd.by/victoria/), но написанная на Python с современным графическим интерфейсом.

Программа позволяет:
- Читать и анализировать SMART-данные дисков
- Оценивать здоровье диска (оценка 0-100)
- Прогнозировать оставшийся ресурс SSD
- Тестировать производительность (бенчмарк)
- Сканировать и лечить поверхность диска
- Экспортировать результаты в файл

### Сравнение с Victoria HDD

| Функция | Victoria HDD | DISK Diagnostic Tool |
|---------|-------------|---------------------|
| SMART чтение | ATA/SATA | ATA/SATA + NVMe + USB-SATA + USB-NVMe |
| Оценка здоровья | Нет | Health Score 0-100, прогноз TBW |
| Сканирование поверхности | Да | Да (Ignore/Erase/Refresh/Write) |
| Drill-down секторов | Нет | Да (4096 байт) |
| Бенчмарк | Базовый | 7 тестов + 4 графика |
| SLC Cache тест | Нет | Да (обнаружение cliff) |
| USB-NVMe мосты | Нет | JMicron/ASMedia/Realtek |
| Двуязычный интерфейс | Нет | Русский + English |
| Тёмная тема | Нет | Catppuccin Mocha |

### Системные требования

- **ОС:** Windows 10/11 или Windows Server 2019+ (64-бит)
- **Права:** Запуск от **Администратора** обязателен
- **Зависимости:** Нет (всё встроено в exe)

### Запуск программы

1. Скачайте `DISK_Diagnostic.exe`
2. Щёлкните правой кнопкой → **"Запуск от имени администратора"**
3. Подтвердите запрос UAC

> ⚠️ Без прав администратора программа не сможет читать SMART и работать с дисками напрямую.

## 3. Вкладка SMART

### Что такое SMART

SMART (Self-Monitoring, Analysis and Reporting Technology) — встроенная система самодиагностики диска. Диск ведёт счётчики ошибок, температуры, времени работы и других параметров.

### ATA/SATA SMART (таблица с 7 колонками)

Для SATA-дисков (HDD и SSD) отображается таблица:

| Колонка | Описание |
|---------|----------|
| **ID** | Номер атрибута (1-254) |
| **Атрибут** | Название (на языке интерфейса) |
| **Текущ** | Текущее значение (чем больше — тем лучше, обычно 100=норма) |
| **Худш** | Худшее зафиксированное значение за всё время |
| **Порог** | Пороговое значение — если Текущ падает ниже, диск считается неисправным |
| **Raw значение** | Сырое значение (реальный счётчик) |
| **Статус** | GOOD / WARN / CRIT |

### NVMe SMART (таблица с 3 колонками)

Для NVMe-дисков отображается:

| Колонка | Описание |
|---------|----------|
| **Параметр** | Название поля (на языке интерфейса) |
| **Значение** | Текущее значение |
| **Статус** | GOOD / WARN / CRIT |

### Цветовая индикация

- **Синий текст** — критический атрибут (влияет на надёжность диска)
- **Зелёный статус** — GOOD, всё в порядке
- **Жёлтый статус** — WARNING, требуется внимание
- **Красный статус** — CRITICAL, серьёзная проблема

### Подсказки

- **Нажмите на строку** — описание атрибута появится внизу таблицы
- **Наведите на Raw Value** — всплывающая подсказка покажет hex-значение и Low16/Low32 (для packed-контроллеров типа SandForce)

### USB-диски

Программа поддерживает SMART через USB:
- **USB-SATA** — через SCSI SAT pass-through (большинство USB-SATA мостов)
- **USB-NVMe** — через vendor-specific протоколы: JMicron, ASMedia, Realtek

## 4. Оценка здоровья

### Health Score (0-100)

Программа рассчитывает оценку здоровья по формуле, учитывающей:

| Фактор | Макс. штраф | Атрибуты |
|--------|------------|----------|
| Переназначенные секторы | -40 | ID 5 |
| Неисправимые ошибки | -40 | ID 187, 198 |
| Ошибки программирования | -30 | ID 171 |
| Ошибки стирания | -30 | ID 172 |
| Ожидающие секторы | -20 | ID 197 |
| Износ SSD | -30 | ID 231, 177 |
| Температура | -15 | ID 194 |
| Ошибки CRC | -10 | ID 199 |
| NVMe Critical Warning | -30 | NVMe |
| NVMe Media Errors | -40 | NVMe |

### Категории оценки

| Оценка | Цвет | Статус | Рекомендация |
|--------|------|--------|-------------|
| 90-100 | Зелёный | ХОРОШО | Всё отлично |
| 70-89 | Зелёный | ХОРОШО | Норма, наблюдайте |
| 50-69 | Жёлтый | ВНИМАНИЕ | Планируйте замену |
| 30-49 | Красный | КРИТИЧНО | Срочная замена |
| 0-29 | Красный | КРИТИЧНО | Немедленная замена |

### TBW Калькулятор

Для SSD показывается:
- **TBW использовано** — сколько терабайт записано на диск
- **TBW номинал** — оценочный ресурс (~600 TB на 1 TB ёмкости для TLC)
- **Запись/день** — среднесуточный объём записи
- **Прогноз** — сколько лет/месяцев/дней осталось при текущем темпе записи

> Если прогноз показывает **"> 100 лет"** — диск используется очень мало для записи. Ресурса хватит надолго.

### Наработка

Отображается в формате: **"1 г 4 мес 12 дн (12,308 ч)"**

> Для контроллеров SandForce (Kingston SKC300 и др.) используется специальная 20-битная маска для корректного извлечения часов из packed raw-значения.

### Особые случаи

- **Мёртвый диск** (0 SMART атрибутов) → "НЕ ОПРЕДЕЛЕНО" вместо ложного "ХОРОШО"
- **HDD** — TBW не отображается (бессмысленно для HDD)
- **Score < 30** — прогноз TBW не показывается (диск при смерти)

## 5. Вкладка Тесты (Benchmark)

### Безопасные тесты (только чтение)

Запускаются всегда, данные не затрагиваются:

| Тест | Что измеряет | Длительность |
|------|-------------|-------------|
| **Послед. чтение** | Скорость последовательного чтения (MB/s) | ~1 сек |
| **4K чтение** | Случайное чтение 4KB — IOPS и задержка (P95/P99) | ~1 сек |
| **Чтение по позициям** | Скорость в 200 точках по всему диску | ~20-60 сек |

### Деструктивные тесты (запись)

Включаются галочкой **"+ Тесты записи"** (красная надпись). **ВСЕ ДАННЫЕ НА ДИСКЕ БУДУТ УНИЧТОЖЕНЫ!**

| Тест | Что измеряет | Длительность |
|------|-------------|-------------|
| **Послед. запись** | Скорость записи (512 MB случайных данных) | ~2 сек |
| **4K запись** | Случайная запись 4KB — IOPS | ~1 сек |
| **Микс 70/30** | 70% чтение + 30% запись случайные 4KB | 30 сек |
| **Проверка** | Запись 256MB → чтение → сравнение MD5 | ~5 сек |
| **SLC кэш** | Непрерывная запись до 50 GB — обнаружение cliff | 1-10 мин |

### 7 карточек результатов

После завершения показываются карточки с большими цифрами:
- Послед. чтение / Послед. запись — MB/s
- 4K чтение / 4K запись — IOPS
- Микс 70/30 — IOPS (R + W отдельно)
- Проверка — ✓ ОК или ✗ ОШИБКА
- SLC кэш — размер кэша в GB или "Без падения"

### 4 вкладки графиков

- **Задержки (точки)** — scatter plot задержек по позиции на диске
- **Гистограмма задержек** — распределение по диапазонам (0-50/50-100/100-200/200-500/500-1K/>1K мкс)
- **Чтение по позициям** — скорость vs позиция (SSD = ровная линия, HDD = убывающая)
- **SLC кэш** — скорость записи vs объём (cliff = точка падения скорости)

### Защита

- Перед write-тестами — предупреждение с указанием **модели диска**
- Для **системного диска** — двойное подтверждение
- При **I/O ошибке** — тест не падает, показывает "✗ Ошибка I/O" в карточке
- Используются `FILE_FLAG_NO_BUFFERING` + `FILE_FLAG_WRITE_THROUGH` для честных результатов
- Случайные данные (не нули) — чтобы контроллер не сжимал

### Экспорт

**Файл → Экспорт бенчмарка (Ctrl+B)** — сохраняет результаты в текстовый файл на языке интерфейса с температурой и всеми метриками.

## 6. Вкладка Поверхность (Surface Scan)

### Блок-карта

Визуализация как в Victoria HDD — сетка цветных прямоугольников:

| Цвет | Категория | Значение |
|------|----------|----------|
| Серый | < 5 мс | Отлично |
| Зелёный | < 20 мс | Хорошо |
| Бирюзовый | < 50 мс | Приемлемо |
| Жёлтый | < 150 мс | Медленно |
| Оранжевый | < 500 мс | Очень медленно |
| Красный | ≥ 500 мс | Критически медленно |
| Крестик X | Ошибка | Сектор не читается |

### Режимы сканирования

#### Ignore (безопасный)
Только чтение. Никаких записей на диск. Показывает карту скорости чтения.

#### Erase
Запись нулей **только в нечитаемые секторы**. Firmware HDD переназначит их из резервной области. Данные в этих секторах уже потеряны.

#### Erase +Slow
Как Erase, но дополнительно стирает медленные секторы (≥150мс). **Данные в медленных секторах будут уничтожены!**

#### Refresh
Чтение → перезапись **тех же данных**. "Освежает" магнитную запись. Счётчик показывает **"Обновлено"** (не "Исправлено"). Данные сохраняются (кроме сбоя питания при записи).

#### WRITE !!! (полное стирание)
Запись нулей на **КАЖДЫЙ** сектор без чтения. **ВСЕ ДАННЫЕ УНИЧТОЖАЮТСЯ!** Все бэд-секторы будут переназначены firmware. Используйте для полного стирания перед утилизацией диска.

### Drill-down

При ошибке чтения блока программа автоматически:
1. Перечитывает блок **посекторно** (по 4096 байт)
2. Находит конкретные битые секторы
3. В режимах Erase/Refresh — пишет нули **только в битые секторы** (хорошие данные не затрагиваются)
4. Показывает LBA битых секторов в реальном времени в скроллируемом списке

### Настройки

- **Блок:** 64 KB / 256 KB / 1 MB (по умолчанию) / 4 MB
- **LBA from / to:** Диапазон сканирования в секторах LBA
- **+ Slow:** Чекбокс для режима Erase +Slow

### Статистика

Справа отображается:
- Счётчики по категориям скорости
- Текущая скорость (MB/s)
- Время и ETA
- Текущая позиция в LBA
- Обновлено/Исправлено + Ошибки записи
- Список битых секторов (LBA)

### Защита

- Перед **WRITE !!!** — предупреждение с моделью диска
- Для **системного диска** — двойное подтверждение
- **Блокировка томов** — автоматическая для всех режимов записи (включая скрытые EFI/Recovery разделы)

## 7. Экспорт данных

### Экспорт SMART (Ctrl+S)

Сохраняет в текстовый файл:
- Информация о диске (модель, серийный номер, прошивка, ёмкость)
- Оценка здоровья (Score, TBW, прогноз, предупреждения)
- Полная таблица SMART-атрибутов (ATA) или NVMe-параметров

Имя файла: `SMART_Model_YYYYMMDD_HHMMSS.txt`

### Экспорт бенчмарка (Ctrl+B)

Сохраняет в текстовый файл:
- Информация о диске
- Все результаты тестов (скорость, IOPS, латентность P95/P99)
- Температура (начало → конец, максимум)

Имя файла: `Benchmark_Model_YYYYMMDD_HHMMSS.txt`

Оба экспорта на языке интерфейса (русский или английский).

## 8. Интерпретация результатов

### SMART: на что обращать внимание

#### Критические атрибуты (выделены синим)

| ID | Атрибут | Что значит raw > 0 |
|----|---------|-------------------|
| 5 | Переназначенные секторы | Диск нашёл bad-секторы и заменил их из резерва |
| 197 | Ожидающие переназначения | Есть нестабильные секторы — могут стать bad |
| 198 | Неисправимые офлайн | Ошибки, которые не удалось исправить |
| 171 | Ошибки программирования | Проблемы с записью в NAND (SSD) |
| 172 | Ошибки стирания | Проблемы со стиранием NAND (SSD) |

#### Информационные атрибуты

| ID | Атрибут | Нормальные значения |
|----|---------|-------------------|
| 9 | Время работы | Любое (просто счётчик часов) |
| 194 | Температура | 25-45°C нормально, >60°C — перегрев |
| 199 | Ошибки CRC | >0 = проблемы с кабелем SATA, НЕ с диском |
| 231 | Остаток ресурса SSD | 100=новый, <10=пора менять |

### Бенчмарк: эталонные значения

| Тест | SATA SSD | NVMe Gen3 | NVMe Gen4 |
|------|----------|-----------|-----------|
| Seq Read | 400-560 MB/s | 2000-3500 MB/s | 5000-7400 MB/s |
| Seq Write | 300-530 MB/s | 1500-3000 MB/s | 3000-7000 MB/s |
| 4K Read QD1 | 8-12K IOPS | 12-20K IOPS | 15-25K IOPS |
| 4K Write QD1 | 20-80K IOPS | 50-200K IOPS | 100-400K IOPS |

### SLC Cache: как читать

- **Ровная линия** — весь тест прошёл на скорости SLC-кэша (кэш > 50 GB)
- **Падение (cliff)** — кэш исчерпан, дальше пишет в NAND напрямую (медленнее)
- **Размер кэша** — точка, где скорость падает

### Drive Sweep: как читать

- **SSD: ровная линия** — нормально, скорость одинакова по всему диску
- **HDD: убывающая кривая** — нормально, внешние дорожки быстрее внутренних
- **Провалы на SSD** — возможен троттлинг или проблемы контроллера

## 9. Устранение проблем

### "SMART не поддерживается"

**Причины:**
- USB-мост не поддерживает pass-through команды
- Диск не отвечает на SMART-запросы
- Недостаточно прав (запустите от Администратора)

### "Отказано в доступе" (ошибка 5)

- Запустите программу от **Администратора**
- Для write-операций: программа автоматически блокирует тома

### "I/O Device Error" (ошибка 1117)

- Диск **физически неисправен** или отключается от шины
- Проверьте кабели и разъёмы
- Для USB — попробуйте другой порт или кабель

### Огромные числа в Raw Value

Контроллеры **SandForce** (Kingston SKC300 и др.) пакуют несколько счётчиков в одно 6-байтное raw-значение. Это нормально. Наведите мышь на значение — подсказка покажет Low16/Low32.

### Программа не запускается на Windows Server 2016

PySide6 6.5+ требует **Windows 10 version 1809** или новее. Windows Server 2016 не поддерживается.

### Иконка программы не обновляется после копирования

Очистите кэш иконок Windows:
```
ie4uinit.exe -show
```
