## Возможности

### Мониторинг SMART
- **ATA/SATA SMART** — все атрибуты, пороги, raw-значения, цветовая индикация
- **NVMe Health Info** — все 16 стандартных полей (температура, резерв, износ, ошибки и т.д.)
- **USB-SATA** — SMART через SCSI SAT pass-through
- **USB-NVMe** — через проприетарные мосты: JMicron JMS583/581, ASMedia ASM2362/2364, Realtek RTL9210/9211/9220
- **Оценка здоровья (0-100)** — взвешенная формула по SSD Testing Spec
- **Калькулятор TBW** — использовано/номинально, суточная запись, прогноз ресурса
- **70+ SMART-атрибутов** — включая Kingston, Samsung, WD, Transcend/Silicon Motion
- **Критические атрибуты** выделены синим цветом
- **Двуязычные названия** атрибутов — русский и английский
- **Экспорт SMART** — Файл → Экспорт SMART (Ctrl+S)

### Бенчмарк (7 тестов)
- **Послед. чтение/запись** — блоки 1 МБ, скорость в МБ/с
- **Случайное 4K чтение/запись** — IOPS, задержка (avg, P95, P99)
- **Микс 70/30** — имитация реальной нагрузки, 30 секунд
- **Проверка записи** — 256 МБ, контроль целостности данных (MD5)
- **Тест SLC-кэша** — непрерывная запись до 50 ГБ, обнаружение cliff
- **Чтение по позициям** — 200 точек по всему диску (скорость vs позиция)
- **Мониторинг температуры** — во время всех тестов
- **4 вкладки графиков** — задержки (точки), гистограмма, чтение по позициям, SLC кэш
- **Прямой ввод-вывод** — `FILE_FLAG_NO_BUFFERING` + `FILE_FLAG_WRITE_THROUGH`
- **Экспорт бенчмарка** — Файл → Экспорт бенчмарка (Ctrl+B)

### Сканирование поверхности (как в Victoria HDD)
- **Ignore** — только чтение, замер задержки по блокам
- **Erase** — запись нулей в нечитаемые секторы (ремап firmware HDD)
- **Refresh** — чтение → перезапись тех же данных
- **WRITE !!!** — полное стирание поверхности (нули на каждый блок)
- **Erase +Slow** — стирание также медленных секторов (≥150мс)
- **Drill-down** — при ошибке блока посекторный анализ (4096 байт)
- **LBA битых секторов** — в реальном времени, скроллируемый список
- **Диапазон LBA** — задать начало/конец для точечного сканирования
- **Блок-карта** — цветная сетка, обновление 30 fps
- **Блокировка томов** — автоматическая перед записью

### Безопасность
- **Защита системного диска** — двойное подтверждение с указанием модели
- **Предупреждения** — все операции записи требуют подтверждения
- **Модель диска** отображается во всех предупреждениях

### Интерфейс
- **Двуязычный** — Русский / English (меню 🌐 Language)
- **Тёмная тема** — Catppuccin Mocha
- **Своя иконка** — диск в стиле Catppuccin

## Требования

- Windows 10/11 или Windows Server 2019+ (64-бит)
- **Права администратора** обязательны
- Python 3.12+ (для разработки)
- PySide6 >= 6.6.0

## Быстрый старт

```bash
python -m pip install PySide6
python run.py  # Запускать от Администратора!
```

## Сборка exe

```bash
python -m pip install pyinstaller
python -m PyInstaller --onefile --windowed --name "DISK_Diagnostic" --icon "disk_diag/resources/app.ico" --clean run.py
```

## Структура проекта

```
disk_diag/
├── core/                   # Бэкенд
│   ├── winapi.py           # CreateFile, DeviceIoControl, ReadFile, WriteFile, блокировка томов
│   ├── smart_ata.py        # ATA SMART: legacy + ATA PT + SCSI SAT
│   ├── smart_nvme.py       # NVMe Health: 5 методов fallback + WMI
│   ├── smart_usb_nvme.py   # USB-NVMe: JMicron/ASMedia/Realtek vendor SCSI
│   ├── health_assessor.py  # Health Score (0-100), TBW, WAF
│   ├── benchmark.py        # 7 тестов бенчмарка + мониторинг температуры
│   └── surface_scan.py     # Сканирование поверхности (Ignore/Erase/Refresh/Write)
├── data/
│   ├── smart_db.py         # 70+ SMART-атрибутов (EN/RU)
│   └── nvme_fields.py      # Описания полей NVMe (EN/RU)
├── gui/
│   ├── main_window.py      # Главное окно, меню, экспорт
│   ├── benchmark_panel.py  # 7 карточек + 4 вкладки графиков
│   ├── surface_panel.py    # Блок-карта + статистика + список битых секторов
│   └── ...                 # info_panel, smart_table, health_indicator, theme
├── i18n.py                 # Локализация: tr("en", "ru")
└── resources/app.ico       # Иконка приложения
```
