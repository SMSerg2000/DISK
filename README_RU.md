## Возможности

### Мониторинг SMART
- **ATA/SATA SMART** — все атрибуты, пороги, raw-значения, цветовая индикация
- **NVMe Health Info** — все 16 стандартных полей (температура, резерв, износ, ошибки и т.д.)
- **USB-SATA** — SMART через SCSI SAT pass-through
- **USB-NVMe** — через проприетарные мосты: JMicron JMS583/581, ASMedia ASM2362/2364, Realtek RTL9210/9211/9220
- **Оценка здоровья (0-100)** — взвешенная формула с защитами от ложных срабатываний
- **Калькулятор TBW/WAF** — использовано/оценка, суточная запись, прогноз ресурса
- **80+ SMART-атрибутов** — включая Kingston, Samsung, WD, Crucial/Micron, Transcend/Silicon Motion
- **Vendor-профили** — корректное декодирование packed raw (SandForce и др.) + переопределение имён по производителю (напр. ID 202 у Crucial = остаток ресурса, а не Address Mark Errors)
- **Виртуальные диски** — распознавание гипервизора (VirtIO/Hyper-V/VMware/Xen/Parallels/GCE/AWS), честное «SMART недоступен»
- **OEM-эвристики** — определение NVMe/SATA за драйверами Intel RST/VMD (bus_type=Unknown)
- **Реальная модель USB-кармана** — через ATA IDENTIFY поверх SAT (вместо «Mass Storage Device»)
- **Критические атрибуты** выделены синим цветом
- **SMART-тренд** — изменение каждого атрибута с прошлой проверки (колонка **Тренд** с ↑/↓) + баннер деградации при росте дефектных счётчиков (Reallocated/Pending/Uncorrectable/CRC; NVMe media-errors / падение spare). Снимки в SQLite
- **Двуязычные названия** атрибутов — русский и английский
- **Экспорт** — SMART (Ctrl+S), бенчмарк (Ctrl+B), JSON (Ctrl+J)

### Бенчмарк (8 тестов)
Только чтение (безопасны, выполняются всегда):
- **Послед. чтение** — блоки 1 МБ, скорость в МБ/с
- **Случайное 4K чтение** — IOPS, задержка (avg, P95, P99, P99.9)
- **Чтение по позициям** — 200 точек по всему диску (скорость vs позиция)

Деструктивные (запись — только в профилях Standard/Full/Stress, **уничтожают данные**):
- **Послед. запись** — блоки 1 МБ, стартует за защищённым 1 ГБ
- **Случайная 4K запись** — IOPS, задержка
- **Микс 70/30** — имитация реальной нагрузки
- **Проверка записи** — 256 МБ (1 ГБ в Stress), контроль целостности (MD5)
- **Тест SLC-кэша** — непрерывная запись до 50/100 ГБ, обнаружение cliff

- **Мониторинг температуры** — фиксируется во время всех тестов
- **4 вкладки графиков** — задержки (точки), гистограмма, чтение по позициям, SLC кэш
- **Прямой ввод-вывод** — `FILE_FLAG_NO_BUFFERING` + `FILE_FLAG_WRITE_THROUGH`
- **Экспорт бенчмарка** — Файл → Экспорт бенчмарка (Ctrl+B)

### Сканирование поверхности (как в Victoria HDD)
- **Ignore** — только чтение, замер задержки по блокам
- **Erase** — запись нулей в нечитаемые секторы (ремап firmware HDD)
- **Refresh** — чтение → перезапись тех же данных
- **WRITE !!!** — полное стирание поверхности (нули на каждый блок)
- **Erase +Slow** — стирание также медленных секторов (≥150мс)
- **Drill-down** — при ошибке блока посекторный анализ (4096 байт)
- **LBA битых секторов** — в реальном времени, скроллируемый список
- **Диапазон LBA** — задать начало/конец для точечного сканирования
- **Блок-карта** — цветная сетка, обновление 30 fps
- **Блокировка томов** — автоматическая перед записью

### Самотест (SMART / NVMe)
- **Короткий / Расширенный** — командует диску запустить встроенную самодиагностику (ATA SMART `EXECUTE OFFLINE` / NVMe Device Self-test)
- **Non-destructive** — диск проверяет сам себя, данные не трогаются
- **Прогресс в реальном времени** — опрос раз в несколько секунд, можно прервать в любой момент
- **Журнал** — прошлые результаты, прочитанные из диска (тип теста, статус, наработка, LBA первой ошибки)
- **Идёт в firmware** — закрытие программы не останавливает запущенный тест (диск продолжает сам)
- **ATA + NVMe + USB-SATA** — прямые SATA/NVMe и USB-SATA мосты; для USB-NVMe честно сообщается, что недоступно

### Журнал ошибок (SMART / NVMe)
- **Журнал ошибок диска** — собственная запись диска о последних сбоях команд (ATA Summary SMART Error Log / NVMe Error Information Log)
- **Расшифровка** — тип ошибки (uncorrectable / aborted / interface-CRC / LBA вне диапазона…), LBA сбоя, наработка (ATA), namespace и код статуса (NVMe)
- **Только чтение, безопасно** — один короткий IOCTL, без записи
- **ATA + NVMe + USB-SATA** — для USB-NVMe честно сообщается, что недоступно

### Безопасность
- **Только чтение по умолчанию** — стандартный путь диагностики (SMART, тесты чтения, режим Ignore) никогда не пишет на диск
- **Запись на raw-диск деструктивна** — любой режим бенчмарка или поверхности, пишущий в `PhysicalDrive`, **перезаписывает данные безвозвратно**
- **Защита зоны MBR/GPT/EFI** — тесты записи обходят первый 1 ГБ, снижая риск уничтожить разметку и загрузчик. Это **не** защищает данные за этой зоной
- **Typed-подтверждение (GUI)** — каждая операция записи (профили записи бенчмарка, surface Erase/Refresh/Write) требует ввести точный серийный номер диска; для системного диска — ввести `DESTROY PHYSICALDRIVE<N>`
- **Fail-closed блокировка томов** — если хоть один том целевого диска не удалось заблокировать и размонтировать, запись отменяется
- **Защита системного диска** — деструктивные операции на диске Windows по умолчанию требуют самого строгого подтверждения
- **CLI: подтверждение по серийнику + TOCTOU-защита** — пересверка диска по серийнику прямо перед записью (на случай переподключения USB)
- **Surface scan учитывает SSD** — для SSD/NVMe режимы «лечения» записью предупреждают, что не лечат флеш и лишь тратят ресурс
- **Антикомпрессия** — несжимаемые случайные данные на каждой итерации записи (снижают искажение от кэша/контроллера на SandForce и др.)

## Известные ограничения

- RAID/HBA-контроллеры могут скрывать или виртуализировать данные SMART/NVMe.
- USB-мосты сильно различаются; некоторые вообще не пропускают SMART/NVMe pass-through.
- Сканирование поверхности HDD-центрично. Для SSD/NVMe это лишь грубая проверка читаемости LBA, **а не** тест NAND-ячеек.
- Тесты записи и режимы записи поверхности работают с raw `PhysicalDrive` и **деструктивны** — перезаписывают данные за защищённым 1 ГБ.
- Health Score эвристичен и **не** заменяет фирменную диагностику производителя.
- Оценка TBW — **прикидка** (эвристика ≈600 TBW/ТБ), пока нет vendor-профиля ресурса; QLC/enterprise сильно отличаются.
- WMI-fallback (за OEM-драйверами Intel RST/VMD) даёт лишь частичные данные NVMe; здоровье показывается как `UNKNOWN`, а не ложный `GOOD`.
- Виртуальные диски по своей природе не отдают физический SMART.
- Самотесты идут через прямые SATA/NVMe и USB-SATA мосты; USB-NVMe мосты обычно не могут запустить тест (честно сообщается).
- Trend-история сравнивает с прошлым снимком **того же** диска (по серийнику); у первого чтения базы для сравнения ещё нет.
- Журнал ошибок показывает последние записи диска (ATA Summary log: последние 5; NVMe: последние 32); USB-NVMe мосты обычно не могут его прочитать.
